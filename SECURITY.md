# Security Policy

## Supported Versions

This is a static marketing website. The table below reflects what receives security attention.

| Component        | Status     |
|------------------|------------|
| Production site  | ✅ Supported |
| `main` branch    | ✅ Supported |

## Reporting a Vulnerability

If you discover a security vulnerability in this site or its codebase, please **do not** open a public GitHub issue.

Instead, report it privately by emailing:

**security@overtimesecurity.com**

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested remediation (optional)

We will acknowledge your report within **48 hours** and aim to resolve confirmed issues within **7 days**.

## Scope

In scope:
- Cross-site scripting (XSS) in forms or dynamic content
- Sensitive data exposure (API keys, credentials)
- Broken access controls
- Dependency vulnerabilities with active exploits

Out of scope:
- Self-XSS
- Issues requiring physical access to a device
- Social engineering attacks
- Theoretical vulnerabilities without a practical exploit path

Thank you for helping keep Over Time Security and its clients safe.
